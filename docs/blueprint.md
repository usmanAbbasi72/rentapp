# **App Name**: Data Keeper

## Core Features:

- Data Entry: Allow users to enter records for money owed, remaining money to others, and daily money use (received/spent, purpose).
- Record Management: Enable users to create, read, update, and delete financial records within the app.
- Firestore Integration: Securely store and retrieve user records using Firebase Firestore database.
- Authentication: Use Firebase Authentication for secure login to the dashboard.
- Usage Analysis: Provide miscellaneous usage analysis to the user.
- Money Track Tools: Offer tools for tracking money owed, money remaining, and daily expenses.
- Financial Suggestions: Provide suggestions for proper money usage. Will use groq api key as a tool.
- Payment Timeline & Alarm: Set up a timeline for paying remaining debts and set alarms as reminders.

## Style Guidelines:

- Primary color: Deep Indigo (#3F51B5) for a sense of trust and reliability.
- Background color: Very light gray (#F5F5F5), almost white.
- Accent color: Soft Violet (#9575CD) as a counterpart for input field highlights and interactive elements.
- Body and headline font: 'Inter', a grotesque-style sans-serif, lending a modern and neutral appearance that fits well with an app handling different types of records.
- Use simple, consistent icons for data categories and actions. Choose icons from a set such as Material Design Icons.
- Implement a clean, card-based layout for displaying records, ensuring readability and easy navigation.
- Subtle transitions when adding, editing, or deleting records to provide a smooth user experience.
